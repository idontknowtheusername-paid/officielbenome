# Règles de Documentation

## INTERDICTION STRICTE

**NE JAMAIS créer de fichiers de documentation (.md) sans demande EXPLICITE de l'utilisateur.**

Cela inclut :
- Fichiers RESUME_*.md
- Fichiers AUDIT_*.md
- Fichiers GUIDE_*.md
- Fichiers RAPPORT_*.md
- Fichiers *_SUMMARY.md
- Fichiers *_PLAN.md
- Fichiers *_README.md (sauf si explicitement demandé)
- Tout autre fichier markdown de documentation

## Exceptions autorisées

Créer un fichier .md UNIQUEMENT si :
1. L'utilisateur demande EXPLICITEMENT "crée un fichier doc" ou "documente ça dans un fichier"
2. C'est un README.md pour un nouveau projet/module (et seulement si demandé)
3. L'utilisateur dit "j'ai besoin d'un guide" ou similaire

## À la place

Quand tu termines une tâche :
- Donne un résumé COURT dans le chat
- Liste les fichiers modifiés si nécessaire
- NE PAS créer de fichier récapitulatif

## Réponse type en fin de tâche

"Terminé. J'ai modifié X fichiers pour corriger Y."

C'est tout. Pas de fichier markdown créé.
